配置Mac OS ssh

设置Mac允许远程登陆

会有错误提示信息，表示当前用户没有权限。这个多半是系统为安全考虑，默认设置的。更改设置如下：进入system preference –> sharing –> 勾选remote login，并设置allow access for all users。再次输入ssh localhost，再输入密码并确认之后，可以看到ssh成功

生成公钥

ssh-keygen -t rsa 一路回车，各种提示按默认不要改，等待执行完毕

ssh-keygen表示生成秘钥；-t表示秘钥类型。这个命令在~/.ssh/文件夹下创建两个文件id_rsa和id_rsa.pub，是ssh的一对儿私钥和公钥。接下来，将公钥追加到授权的key中去

然后执行： ls ~/.ssh 可以看到两个密钥文件：id_rsa（私钥）id_rsa.pub（公钥）

变量mapred.job.tracker 保存了JobTracker的位置，因为只有MapReduce组件需要知道这个位置，所以它出现在mapred-site.xml文件中。

编辑 hdfs-site.xml

变量dfs.replication指定了每个HDFS数据库的复制次数。 通常为3, 由于我们只有一台主机和一个伪分布式模式的DataNode，将此值修改为1。
